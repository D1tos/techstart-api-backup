# Работа с Git: краткая выжимка

## 📦 Создание репозитория

Существует **2 способа начать работу с Git**:

### 1. Инициализация нового репозитория
Используется, когда:
- начинаешь новый проект
- добавляешь Git в уже существующий проект

**Команда:**
```bash
git init

📌 Что происходит:

создаётся папка .git (служебная)
формируется пустой репозиторий

⚠️ Важно:

После git init файлы ещё не отслеживаются — их нужно добавить вручную.

2. Клонирование репозитория

Используется, когда:

нужно скачать готовый проект (например, с GitHub)

Команда:

git clone <url>

📌 Что происходит:

создаётся папка проекта
загружается вся история коммитов
появляется рабочая версия файлов

📡 Поддерживаемые протоколы:

HTTPS (самый простой)
SSH (через ключи)
Git / локальный путь
⚙️ Базовая настройка

Перед работой нужно указать автора коммитов:

git config --global user.name "Имя Фамилия"
git config --global user.email "email@example.com"

📌 --global — настройки применяются ко всем репозиториям

🔄 Рабочий процесс Git
Три области

Git работает с тремя зонами:

Рабочая директория — редактирование файлов
Индекс (staging area) — подготовка изменений
Репозиторий — сохранённые коммиты
📁 Состояния файлов

Файл может находиться в одном из состояний:

Untracked — не отслеживается
Unmodified — без изменений
Modified — изменён
Staged — подготовлен к коммиту
🔁 Жизненный цикл файла
Untracked → Staged → Commit → Unmodified
                  ↓
              Modified → Staged
🔧 Типичный цикл работы
✏️ Изменить файлы
🔍 Проверить статус
➕ Добавить в staging
💾 Сделать коммит
🔁 Повторить

📌 Это позволяет делать логичные и понятные коммиты

🗂️ Структура репозитория (.git)

Основные элементы:

objects/ — база данных Git
refs/ — ветки и теги
HEAD — текущая ветка
index — staging area
config — настройки
hooks/ — скрипты автоматизации

⚠️ Не редактируй .git вручную — используй команды Git

🚀 Первые шаги после создания

После создания репозитория обычно делают:

📄 README.md — описание проекта
🚫 .gitignore — исключения файлов
💾 первый коммит
🌐 подключение удалённого репозитория
💡 Итог

Git работает как цикл:

изменения → подготовка → коммит → повтор
