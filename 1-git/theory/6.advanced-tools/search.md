# 🔍 Поиск в Git: log, grep, blame, bisect — краткая выжимка

## 📌 Зачем это нужно
Git — это не только контроль версий, но и мощный инструмент анализа:
- кто написал код?
- когда появился баг?
- где используется функция?

---

# 📜 Git Log — поиск по истории

## 🔎 Базовый поиск
```bash
git log
git log --oneline
👤 По автору и времени
git log --author="John" --since="1 month ago"
git log --after="2023-01-01" --before="2023-06-30"
💬 По сообщениям коммитов
git log --grep="fix"
🔍 По изменениям в коде
По строке (точное изменение)
git log -S"calculateTax"
По regex (любые изменения)
git log -G"function.*calculate" --patch

👉

-S → добавление/удаление строки
-G → любые изменения по regex
📂 По файлам
git log -- path/to/file.js
⚡ Git Grep — поиск по коду
📌 Что делает

Ищет в содержимом файлов, а не в истории

🔎 Примеры
git grep "TODO"
git grep "API_KEY" v2.0.0
🌍 Поиск во всех ветках
git grep "TODO" $(git rev-list --all)
🚀 Преимущества
быстрее обычного grep
игнорирует .gitignore
работает с любыми коммитами
🧑‍💻 Git Blame — кто написал строку
📌 Что показывает
автора строки
коммит
дату
git blame file.js
💡 Когда использовать
понять контекст кода
найти автора изменения
разобраться в логике
⚠️ Особенность

Показывает последнего изменившего, а не автора идеи

🚫 Игнор форматирования

Можно исключить косметические коммиты:

.git-blame-ignore-revs
🧠 Git Bisect — поиск бага
📌 Что делает

Находит коммит с багом через бинарный поиск

🚀 Использование
git bisect start
git bisect bad HEAD
git bisect good v1.0.0

Далее:

git bisect good
# или
git bisect bad
⚙️ Как работает
делит историю пополам
проверяешь коммит
повторяет до нахождения

👉 O(log n) вместо O(n)

🤖 Автоматизация
git bisect run npm test

✔ 0 → good
❌ ≠0 → bad

⏭ Пропуск коммитов
git bisect skip
🔗 Комбинирование инструментов
🐞 Поиск бага
bisect → найти коммит
blame → понять код
log → изучить изменения
grep → найти использование
🏺 Археология кода
blame → кто и когда
log -S → когда добавили
grep → где используется
💡 Практические советы
Используй алиасы для частых команд
Настраивай формат вывода (--oneline, --pretty)
Комбинируй с grep, awk, sed
Документируй найденные инсайты
🧾 Коротко

log — ищешь в истории
grep — ищешь в коде
blame — кто писал
bisect — кто сломал
