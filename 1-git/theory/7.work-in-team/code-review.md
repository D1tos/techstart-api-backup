# Code Review: best practices

## 📌 Коротко
**Code review** — это проверка кода перед merge:
- поиск ошибок
- улучшение качества
- обмен знаниями

👉 Это не критика человека, а улучшение кода

---

## 🎯 Зачем нужен review

- 🐛 Найти баги до production  
- 📚 Распространить знания в команде  
- 🧩 Поддерживать единый стиль  
- 🔒 Повысить безопасность кода  

---

## 📦 Подготовка Pull Request

- ✅ 200–400 строк — оптимально

👉 Code review — это инструмент роста, а не контроль
