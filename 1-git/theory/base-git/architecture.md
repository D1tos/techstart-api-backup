# Философские принципы Git

## Распределенность как основа
Каждая копия полноценна и самодостаточна

## Целостность данных превыше всего
Использование криптографических хеш-функции для обеспечение целостности.
невозможно изменить любую часть истории без изменения последующих идентификаторов

## Произзводительность для типичных операций
Повседневная работа должна быть мгновенной для операций, которые используются постоянно

## Простота реализаций
всего 4 типа объектов и несколько текстовых файлов для ссылок.

# Архитектура Git

## Как Git хранит данные
Git сохраняет:
-Содержимое каждого файла
-Структуру папок проекта
-Информацию о каждом изменении (коммите)
-Связи между изменениями

Каждый сохраненный объект получает уникальный идентификатор.
Изменение содержимого = изменение кода
Данные не могут быть подменены незаметно

## Три рабочие области

### Рабочая папка (git add)
Обычная папка на диске, где лежат файлы проекта.
Вся работа тут: Редактирование, создание и удаление файлов.

### Область подготовки (индекс) (git commit)
Промежуточное место, где собираются изменения для следующего коммита.
Позволяет выбрать какие именно изменения созранить в истории

### Репозиторий (git checkout)
Место в папке .git
Хранится вся история проекта.
Попадают изменения после создания коммита

## История как цепочка изменений
История проекта - цепочка коммитов.
Каждый коммит знает какой коммит родительский.
1-ый коммит не имеет родителя
слияние веток - коммит с несколькими родителями

+Невозможно изменить старый коммит без изменения всех последующих. защищает от подделок 
+Легко найти общую точку при слиянии веток. Можно вернуться к любому состояния проекта
