# Что такое система контроля версий
(Version Control System, VCS) Система контроля версий)

ПО для управлениями изменений файлов. Фиксирует каждое изменение в репозитории, сохраняя содержимое файлов, автора изменений, когда это было и какая была цель
Помимо восстановления предыдущего состояние, можно проследить эволюцию каждой строки кода, констект изменений и эффективно работать с параллельными линиями разработки

## Основные возможности VCS
Решает 4 проблемы разработки

### Проблема потери данных
Любое изменение, зафиксированное в VCS сохраняется навсегда.
механизм коммитов(коммит - неизменяемый снимок состояниа проекта)

### Проблема командной работы
VCS автоматически отслеживает все изменения даже в один и тех же файлах.
Предоставляет механизмы объединения изменений. если конфликт - инструменты для разрешения 

### Проблема аудита изменений
Каждый коммит связан автором и снабжен коммментарием. 
Понимаем Что было изменено,почему изменено. Крит.важно при поиске причин ошибок и логике реализации

### Проблема экспериментальной разработки  
Создание изолированных веток разработки. Не влияет на стабильную версию проекта.
могут быть интегрированы в основную ветку или отброшены без последствий

## Архитектурные подходы
3 основых подхода к построению VCS

### Локальные VCS
Хранит всю информацию о версиях на 1 компьютере.
RCS (Revision Control System) сохранение изменений в виде набора патчей.
найти опред. версию файла, система последовательно применяет патчи к базовой версии

+Эффективен для индивидуальной работы
-Не поддерживает совместную разработку
-Создает риски потери всей истории при сбое локального хранилища 

### Централизованные системы (CVCS)
Subversion (SVN) / Perforce. Архитектура клиент-сервер.
Единственный сервер хранит все версии файлов. Клиенты получают только актуальные.
Требуется подключение к серверу для большинства операций

+Простота администрирования
+Четкий контроль доступа
-зависимость от доступности сервера и производительности сети
Сервер - единая точка отказа

### Распределенные системы (DVCS)
Git Mercurial
Каждый клмент содержит полную копию репозитория со всей историей.
Большинство операций выполняется локально. Синхронизация только при необходимости обмена изменениями

+обеспечивает высокую скорость работы
+возможность автономной разработки 
+естественное резервирование - каждая копия репозитория является полноценным бэкапом

## Git - Стандарт VCS
Принципы:

### Скорость
Большинство операций должны выполняться локально без обращения к сети

### Простота дизайна
Использование простых структур данных и алгоритмов

### Сильная поддержка нелинейной разработки
Эффективное создание и слияние тысяч параллельных веток

### Полная распределенность
Отсутствие выделенного центрального сервера

### Масшабируемость
Способность эффективно работать с большими проектами

Любое изменение в истории будет немедленно обнаружено
