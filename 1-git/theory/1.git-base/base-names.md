# Репозиторий, коммит, ветка — база Git

## Кратко
- **Репозиторий** = папка проекта + скрытая папка `.git` (вся история)
- **Коммит** = неизменяемый снимок проекта + автор + описание + хеш
- **Ветка** = лёгкий указатель на коммит (не копия файлов)
- **HEAD** = указатель «где я сейчас»

> Эти три понятия — фундамент Git. Всё остальное — их комбинации.

## 1. Репозиторий

### Что это физически
Обычная папка проекта, внутри которой есть скрытая папка **`.git`**.

project/
├── .git/ ← ВСЯ история и служебные данные
│ ├── objects/ ← все версии файлов
│ ├── refs/ ← ветки и теги
│ ├── HEAD ← где вы сейчас
│ └── config ← настройки
├── src/ ← ваши рабочие файлы
└── README.md
text


> Удалишь `.git` — история потеряна навсегда. Сами файлы останутся, но версионирование исчезнет.

### Два типа репозиториев

| Тип | Что содержит | Где используется |
|-----|--------------|------------------|
| **Обычный** | историю + рабочие файлы | локальная разработка |
| **Голый (bare)** | только историю (без рабочих файлов) | серверы (GitHub, GitLab) |

## 2. Коммит

### Что содержит каждый коммит
- **Полный снимок** всех файлов проекта (не только изменения!)
- Уникальный идентификатор — **хеш** (например, `a5f3b8c`)
- Автор + дата + время
- Сообщение коммита (что и почему)
- Ссылка на **родителя** (предыдущий коммит)

### Пример

commit a5f3b8c7d9e2f1a4b6c8d0e3f5a7b9c1d3e5f7a9
Author: Anna Petrova anna@example.com
Date: Mon Mar 15 10:30:00 2024 +0300

Исправлена ошибка валидации в форме регистрации
text


### Ключевое свойство — неизменяемость
> Коммит нельзя изменить. Даже исправить опечатку в описании = создать **новый** коммит с **новым** хешем.

### История как цепочка
Каждый коммит знает своего родителя:

[К1] ← [К2] ← [К3] ← [К4]
text


## 3. Ветка

### Что это технически
**Именованный указатель на коммит** — просто текстовый файл с хешем коммита.

> Никакого копирования файлов при создании ветки не происходит.

### Характеристики
- Создание ветки — мгновенно и дёшево
- Новый коммит → ветка автоматически двигается на него
- Веток можно иметь сколько угодно
- Переключение между ветками **меняет содержимое рабочей папки**

### Стандартные имена
- `main` или `master` — основная ветка по умолчанию

### HEAD — указатель текущего положения

HEAD → main → a5f3b8c
text

HEAD обычно указывает на **ветку**, а ветка — на **коммит**.

## Как всё работает вместе (пример)

[C3] ← [C4] ← feature (ветка для новой фичи)
/
[C1] ← [C2] ← main
text


**Процесс:**
1. В репозитории есть `main` на коммите C2
2. Создаётся ветка `feature` — она тоже указывает на C2
3. В `feature` делаются коммиты C3, C4
4. `feature` автоматически двигается на C4, `main` остаётся на C2
5. После завершения — изменения из `feature` вливаются в `main`

## Практическое значение

| Концепция | Что даёт понимание |
|-----------|---------------------|
| **Репозиторий** | Знаете, где история (`.git`) и почему её потерять легко |
| **Коммиты неизменяемы** | Не пытайтесь «исправить прошлое» — будет новый коммит |
| **Ветка — это указатель** | Не бойтесь создавать/удалять ветки — это дёшево |

## Итоговая схема

Репозиторий = контейнер
└── Коммиты = снимки состояний (связаны в цепочку)
└── Ветки = именованные наклейки на коммиты
└── HEAD = наклейка «я сейчас тут»
