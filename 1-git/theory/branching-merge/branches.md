# Ветки: создание, переключение, удаление
Позволяет вести параллельную разработку различных функций

## Концепция
Ветка - подвижный указатель на определенный коммит.
Операции с ветками выполняются мгновенно не завимо от размера проекта

## Создание веток
Создает новую ветку,не переключается на нее:
$ git branch feature-login

Создает ветку и сразу переулючается
$ git switch -c feature-login
-с - флаг создания

Классический способ создание и переключение
git checkout -b

## Переключение между ветками
Основная команда
git switch
выдаст предупреждение с незафиксированными изменениями и откажется выполнять операцию
решения:
- коммит
- временное сохранение git stash
- отмена изменений

Полезность:
переключение на предыдущую ветку с помощью дефиса

## Просмотре игформации о ветках
git branch
без параметров - список веток. * - текущая ветка
флаг -v добавляет последний коммито каждой ветки.
--merged
--no-merged
Показывают слиты ветки или нет
-a просмотр удаленных веток

## Удаление веток
git branch -d
удаляет если полностью слита с основой
-D Принуждение к удалению
Удаление в удалённом репозитории - push со специальным синтаксисом
Удалил - очисти локальные ссылки git remote prune

## Практические аспекты работы с ветками
Ветки создаются для логически обособленных задач-
новые функции, исправления бага, подготовка к релизу.

Поддерживать актуальность веток относительно основной линии.

Визуализация структуры веток
git log --graph --oneline
