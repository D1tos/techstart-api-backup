# Разрешения конфликтов
## Природа конфликтов в Git
### Конфликт слияния
автоматически объединения невозможно
Конфликт возникает только при изменении одних и тех же строк или 
при противоречивых структурных изменениях.

## Типы конфликтов
### Конфликты содержимого
ветки изменяют один и теже строки файла
### Конфликт удаление
один удаляет, другой изменяет
### Конфликты переименования
переименовании файла в разные имена в разных ветках
1 переименовывает, 2-ой изменяет содержимое под старым именем

## Процесс разрешения конфликтов
### 1.Идектификация конфликтов
При возникновении конфликта Git выводит информацию о проблемных файлах
Команда git status показывает детальную информацию о состоянии

### 2.Анализ конфликтов
Git помечает конфликтующие участки специальными маркерами прямо в файле
Структура маркеров конфликта:

    <<<<<<< HEAD — начало блока из текущей ветки
    ======= — разделитель между версиями
    >>>>>>> feature/discount — конец блока из сливаемой ветки

### 3.Принятие решения
Варианты решения:
    Оставить версию из текущей ветки
    Принять версию из сливаемой ветки
    Объединить оба варианта
    Написать совершенно новый код

### 4.Редактирование файла
После принятия решения нужно отредактировать файл
удалив маркеры конфликта и оставив нужный код

### 5.Завершение слияния
После разрешения всех конфликтов необходимо:
#### Добавить разрешённые файлы в индекс
$ git add src/calculator.py

#### Завершить слияние
$ git commit

## Инструменты для работы м конфликтами
### Визуальные инструменты
#### Запуск визуального инструмента
$ git mergetool

#### Настройка предпочитаемого инструмента
$ git config --global merge.tool vscode
$ git config --global mergetool.vscode.cmd 'code --wait $MERGED'

### Команды для анализа
#### Показать базовую версию файла
$ git show :1:src/calculator.py
#### Показать версию из текущей ветки
$ git show :2:src/calculator.py
#### Показать версию из сливаемой ветки
$ git show :3:src/calculator.py
#### Визуализация конфликта в формате diff3
$ git checkout --conflict=diff3 src/calculator.py

## Специальные случаи
### Конфликты при rebase
При rebase конфликты разрешаются для каждого коммита отдельно:
#### После разрешения конфликта при rebase
$ git add .
$ git rebase --continue
#### Пропуск проблемного коммита
$ git rebase --skip
#### Отмена rebase
$ git rebase --abort

### Конфликты в бинарных файлах
Для бинарных файлов (изображения, архивы) Git не может показать различия. Нужно выбрать одну из версий целиком:
#### Выбрать нашу версию
$ git checkout --ours path/to/file.jpg
#### Выбрать их версию
$ git checkout --theirs path/to/file.jpg

## Стратегии слияния
Указывает автоматически разрешение определённых типов конфликтов:
#### Автоматически выбирать "нашу" версию при конфликтах
$ git merge -X ours feature/branch
#### Автоматически выбирать "их" версию
$ git merge -X theirs feature/branch

## Откат неудачного слияния
#### Отмена слияния в процессе
$ git merge --abort
#### Откат уже завершённого слияния
$ git reset --hard HEAD~1
