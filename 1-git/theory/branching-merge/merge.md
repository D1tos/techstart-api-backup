# Слияние веток: fast-forward 3-way merge
Интеграция изменений в основую ветвь

## Концепция слияния веток
объединение 2 и более историй разработки. 
система анализирует изменения в обеих ветках
создает новое сосотяние(включает все источники)

находит общего предка веток и определяет оптимальный способ объединения 

### fast-forward merge
самый простой тип.
Целевая ветка не содержит новых коммитов после точки отвлетвления.
история остается линейной
нет истории изменений ветки
#### Предотвращение fast-forward
git merge --no-ff feature
создает явную точку в истории

### 3-way merge
обе ветки развивались независимо после точки расхождения
3 точки: общий предок и конечные состояния веток

          C---D---E  feature
         /
    A---B---F---G    main

Обе ветки имеют уникальные коммиты после точки B. При слиянии Git:

    Находит общего предка (коммит B)
    Определяет изменения в feature (B→E)
    Определяет изменения в main (B→G)
    Объединяет оба набора изменений
    Создаёт новый коммит слияния

$ git checkout main
$ git merge feature
Merge made by the 'ort' strategy.
 feature.txt | 25 +++++++++++++++++++++++++
 1 file changed, 25 insertions(+)

          C---D---E
         /         \
    A---B---F---G---H

## Стратегия слияния
ort - по умолчанию
ours - коммит слияния,игнорит изменения из веток. закрытие старых веток с сохранением истории
$ git merge -s ours obsolete-feature
octopus - сливание 2+ веток одновременно. Автоматически выбирает при таком сценарии

## Процесс слияния
- Переключаемся на целевую ветку
- merge 

перед слиянием убеждаемся в чистоте дироектории.

## Просмотр результатов слияния
git log --graph
git show слияние относительно всех родителей
флаг --first-parent только с основной линии

## Отмена слияния
git reset
$ git reset --hard HEAD~1

для опубликованных коммитов
git revert
$ git revert -m 1 HEAD
Флаг -m 1 указывает, какого родителя считать основным при отмене слияния.

## Рекомендации
Стратегий зависит от принятой в команде
перед слиянием просмотреть изменения git log с указанием диапозонов 2 коммитов
регулярно сливать в основу
