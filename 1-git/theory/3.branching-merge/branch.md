# Ветки: создание, переключение, удаление

📌 Особенности:
- не копирует файлы
- занимает ~41 байт
- двигается вперёд при новых коммитах

👉 Поэтому:
- создание ветки — мгновенно  
- переключение — быстро даже в больших проектах  

---

## 🌱 Создание веток

### Создать ветку (без переключения)
```bash
git branch feature-login
Создать и сразу перейти
git switch -c feature-login
Старый способ
git checkout -b feature-login
📛 Именование веток

Популярный стиль:

feature/... — новая функция
bugfix/... — исправление
hotfix/... — срочно
release/... — релиз
🔄 Переключение веток
git switch main

📌 Что происходит:

перемещается HEAD
обновляется индекс
меняются файлы в проекте
🔁 Быстрое переключение назад
git switch -
⚠️ Важно

Git не даст переключиться, если есть конфликтующие изменения.

👉 Варианты:

сделать коммит
сохранить изменения (git stash)
отменить изменения
📊 Просмотр веток
git branch
С деталями:
git branch -v
Полезные флаги:
git branch --merged     # уже слиты
git branch --no-merged  # не слиты
git branch -a           # все (включая удалённые)
🗑️ Удаление веток
Безопасное удаление
git branch -d feature-login

📌 Работает только если ветка уже слита

Принудительное удаление
git branch -D feature-login

⚠️ Может привести к потере данных

Удаление удалённой ветки
git push origin --delete feature-login
Очистка ссылок
git remote prune origin
🧠 Практика работы с ветками
✅ Хорошие привычки
Создавай ветку под каждую задачу
Делай ветки короткоживущими
Регулярно обновляй ветку (merge / rebase)
Удаляй неактуальные ветки
⚠️ Проблемы долгих веток
конфликты при слиянии
сложная интеграция
устаревший код
🌳 Визуализация
git log --graph --oneline --all

📌 Помогает увидеть:

разветвления
слияния
структуру проекта
🔗 Связь с удалённым репозиторием

Важно настроить upstream:

git push -u origin feature-login

📌 Это позволяет:

отслеживать изменения
удобно делать push / pull
💡 Итог

Ветки = основа Git:

изолируют изменения
упрощают разработку
позволяют работать параллельно

📌 Главное правило:

одна задача = одна ветка
