# Слияние веток в Git (merge)

После работы в ветке изменения нужно **объединить с основной веткой**.  
Для этого используется `git merge`.

---

## 🔀 Что такое merge

Слияние — это:
> объединение истории двух веток с сохранением всех изменений

📌 Git:
- находит общий коммит (предка)
- анализирует изменения
- объединяет их

---

## ⚡ Типы слияния

Git использует 2 основных типа:

- **Fast-forward**
- **Three-way merge**

---

## 🚀 Fast-forward merge

### 📌 Когда происходит
Если в целевой ветке **нет новых коммитов**

### Пример:


main: A---B

feature: C---D---E


### Команда:
```bash
git checkout main
git merge feature
Результат:
main:    A---B---C---D---E

📌 Особенности:

нет merge-коммита
история линейная
просто "перемотка" ветки вперёд
⛔ Как запретить fast-forward
git merge --no-ff feature

📌 Зачем:

сохранить факт слияния
видеть границы фичи в истории
🔀 Three-way merge
📌 Когда происходит

Если обе ветки изменялись

Пример:
          C---D---E  feature
         /
A---B---F---G        main
Команда:
git checkout main
git merge feature
Результат:
          C---D---E
         /         \
A---B---F---G---H   (merge commit)

📌 Особенности:

создаётся новый коммит (H)
имеет 2 родителя
сохраняет обе истории
⚙️ Стратегии слияния
По умолчанию:
ort (новые версии Git)
recursive (старые)
Полезные стратегии:
# Игнорировать изменения второй ветки
git merge -s ours branch

# Слить несколько веток сразу
git merge branch1 branch2 branch3
🔄 Процесс слияния
Перейти в целевую ветку:
git switch main
Выполнить merge:
git merge feature
⚠️ Перед merge

Обязательно:

git status

📌 Рабочая директория должна быть чистой

📊 Просмотр слияний
git log --graph --oneline

📌 Показывает:

ветки
merge-коммиты
структуру истории
Показать merge-коммит:
git show <commit>
❌ Отмена слияния
Если не запушено:
git reset --hard HEAD~1
Если уже опубликовано:
git revert -m 1 HEAD

📌 -m 1 — основной родитель

🧠 Практика
Когда использовать fast-forward:
простая история
маленькие проекты
Когда использовать merge-коммиты:
командная разработка
важно видеть фичи
💡 Советы
Делай merge регулярно
Не держи ветки долго
Проверяй изменения перед merge:
git log main..feature
🧾 Итог

Merge — это способ объединения разработки:

fast-forward → линейная история
3-way → сохраняет структуру веток

📌 Главное:

чем раньше объединяешь ветки — тем меньше конфликтов
