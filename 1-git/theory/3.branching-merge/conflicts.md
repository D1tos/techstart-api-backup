# Разрешение конфликтов в Git

Конфликты — это нормальная часть командной работы.  
Они возникают, когда Git **не может автоматически объединить изменения**.

---

## ⚠️ Что такое конфликт

Конфликт = ситуация, когда:
> изменения затрагивают одни и те же строки

Git сравнивает:
- 🧱 **Базу** (общий предок)
- 🟢 **Ours** — текущую ветку  
- 🔵 **Theirs** — сливаемую ветку  

---

## 🔥 Когда возникают конфликты

### 1. Конфликт содержимого
Изменены одни и те же строки

---

### 2. Удаление vs изменение
- один удалил файл  
- другой изменил  

---

### 3. Переименование
- разные имена  
- или rename + изменение  

---

🖼️ Бинарные файлы

Git не может сравнивать:

# выбрать нашу версию
git checkout --ours file.jpg

# выбрать их версию
git checkout --theirs file.jpg
⚙️ Автоматические стратегии
# всегда брать нашу версию
git merge -X ours branch

# всегда брать их версию
git merge -X theirs branch
