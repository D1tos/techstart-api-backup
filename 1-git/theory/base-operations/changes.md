# Работа с изменениями (add, commit, status, diff)

## Проверка состояния — git status
Показывает текущее состояние рабочей директории и области подготовки
какие файлы изменены, что готово к коммиту, в какой ветке происходит работа

Вывод команды разделен на секции:
-Текущая ветка		Активная ветка
-Изменения не в индексе	Файлы изменены, но не подготовлены к коммиту
-Неотслеживаемые файлы	Новые файлы, о которых не знает гит
-Изменения в индексе	Файлы, готовые к коммиту

Краткая форма вывода флаг -s
Символы в первой колонке показывают состояние в индексе
Во второй — в рабочей директории 
Символ M означает модификацию 
?? — неотслеживаемый файл.

## Просмотр изменений — git diff
Показывает конкретные изменения в файлах

Вывод показывает:
-Какие файлы сравниваются
-Индексы файлов в гит
-Изменения: строки с - удалены, с + добавлены

Варианты:
### Изменения в рабочей директории (ещё не в индексе)
$ git diff

### Изменения в индексе (подготовленные к коммиту)
$ git diff --staged

### Изменения в конкретном файле
$ git diff README.md

### Сравнение с конкретным коммитом
$ git diff HEAD~1

## Добавление в индекс — git add
Переном измененеия из рабочей директории в область подготовки.
Точно контролирует, что будет зафиксировано

### Добавить конкретный файл
$ git add README.md

### Добавить несколько файлов
$ git add file1.txt file2.txt

### Добавить все файлы в текущей директории
$ git add .

### Добавить все изменения в репозитории
$ git add -A

### Интерактивное добавление частей файла
$ git add -p

После файл переходит в staged, будет включен в след коммит
в индеки добавляется текущее содержимое файла.
если файл будет изменен снова, новые изменения не попадут
работает как с новыми так и измененными файлами

## Создание коммита — git commit
Фиксирует состояние проекта в истории

### Коммит с сообщением в командной строке
$ git commit -m "Add user authentication feature"
[main a5f3b8c] Add user authentication feature
 2 files changed, 45 insertions(+), 3 deletions(-)

### Коммит с открытием редактора для сообщения
$ git commit

### Коммит всех изменённых отслеживаемых файлов
$ git commit -am "Update configuration files"

### Изменение последнего коммита
$ git commit --amend

Стуктура хорошего сообщения коммита:
-Описание изменений(до 50 символов)

Каждый коммит получает уникальный SHA-1 идентификатор и содержит:

Снимок всех файлов проекта
Ссылку на родительский коммит
Информацию об авторе и времени
Сообщение с описанием изменений

## Рабочий процесс с изменениями
### 1. Проверка текущего состояния
$ git status

### 2. Просмотр конкретных изменений
$ git diff

### 3. Добавление нужных файлов в индекс
$ git add src/auth.js src/auth.test.js

### 4. Проверка что будет включено в коммит
$ git diff --staged

### 5. Создание коммита
$ git commit -m "Implement user authentication"

### 6. Проверка что коммит создан
$ git log -1

## Частичное добавление изменений
Добавить только часть изменений из файла.

### Интерактивный режим добавления
$ git add -p filename.js

В интерактивном режиме Git показывает каждое изменение и предлагает варианты:

y — добавить изменение в индекс
n — пропустить изменение
s — разбить изменение на более мелкие части
e — редактировать изменение вручную
