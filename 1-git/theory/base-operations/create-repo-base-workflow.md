# Создание репозмтория

## Основные подходы получить репозитории
### Инициализация нового репозитория
Новый проект/Добавить VCS уже к существующему проекту - инициализация
git init
создает следующую пустую инфраструктуру для работы

project/
├── .git/          # Служебная директория Git
│   ├── HEAD       # Указатель на текущую ветку
│   ├── config     # Конфигурация репозитория
│   ├── objects/   # База данных объектов
│   └── refs/      # Ссылки на коммиты
└── ... файлы проекта ...

### Клонирование существующего репозитория
Копия существующего проекта - клонирование
git clone
загружает полную историю проекта и создает рабочую копию

Происходит:
-Создаётся новая директория с именем проекта
-Инициализируется Git-репозиторий внутри неё
-Загружается вся история коммитов
-Создаётся рабочая копия последней версии файлов

## Базовая конфигурация

git config --global user.name "Имя Фамилия"
git config --global user.email "email@example.com"

### Концепция рабочего процесса
У Git определенный цикл, который повторяется при каждом изменении проекта

#### Три состояния
-Рабочая директория
-Область подготовки(индекс)
-Репозиторий

#### Жизненный цикл файлов
Неотслеживаемый (Untracked) — Git не знает об этом файле
Отслеживаемый неизменённый (Unmodified) — файл под контролем версий, не изменялся с последнего коммита
Изменённый (Modified) — файл был изменён после последнего коммита
Подготовленный (Staged) — изменения подготовлены для включения в следующий коммит

#### Типичный рабочий процесс
1.Внесение изменений — редактирование файлов в рабочей директории
2.Просмотр состояния — проверка, какие файлы изменены
3.Подготовка изменений — выбор файлов для включения в коммит
4.Фиксация изменений — создание коммита с описанием
5.Повторение цикла — возврат к пункту 1

## Структура Git-репозитория
Основные компоненты директории .git

objects/ — база данных всех объектов (файлов, коммитов, деревьев)
refs/ — ссылки на объекты коммитов (ветки, теги)
HEAD — указатель на текущую ветку
index — информация об области подготовки
config — конфигурация репозитория
hooks/ — скрипты для автоматизации

## Первые шаги после создания репозитория
Создание файла README с описанием проекта
Добавление файла .gitignore для исключения ненужных файлов
Создание первого коммита с начальной структурой проекта
Настройка удалённого репозитория (при необходимости)
