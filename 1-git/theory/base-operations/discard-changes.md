# Отмена изменений (checkout, restore, clean)
## Эволюция команд отмены
git checkout - историческая команда
git restore - восстановление файлов
git switch - переключение веток

## Восстановление файлов - git restore
Восстановление файлов в рабочей директории.
основной инструмент для отмены изменений

### Отмена изменений в рабочей директории
Отмена незакоммиченных изменений в файле
#### Восстановить файл до состояния последнего коммита
$ git restore README.md
#### Восстановить несколько файлов
$ git restore file1.txt file2.txt
#### Восстановить все файлы в директории
$ git restore src/
#### Восстановить все изменённые файлы
$ git restore .
Файлы вернутся к состоянию последнего коммита

### Отмена изменений в индексе
Отмена изменений командной git add
#### Убрать файл из индекса, сохранив изменения в рабочей директории
$ git restore --staged README.md
#### Убрать все файлы из индекса
$ git restore --staged .
#### Убрать из индекса и отменить изменения в рабочей директории
$ git restore --staged --worktree README.md

### Восстановление из конкретного коммита
Восстановление файлов из любого другого
#### Восстановить файл из конкретного коммита
$ git restore --source=HEAD~2 README.md
#### Восстановить файл из другой ветки
$ git restore --source=feature-branch config.json
#### Восстановить удалённый файл
$ git restore --source=HEAD~1 deleted-file.txt

## Классический подход - git checkout
широко используется, особенно в легаси проектах

#### Отменить изменения в файле (аналог git restore)
$ git checkout -- README.md
#### Восстановить файл из конкретного коммита
$ git checkout HEAD~2 -- README.md
#### Восстановить все файлы
$ git checkout -- .

'--' отделяет имена файлов от других параметров команды, предотвращая неоднозначность

## Удаление неотслеживаемых файлов - git clean
Удаляет файлы, которые не находятся под контролес версий.
Очистка от временных файлов, результаты сборки и другого мусора.

#### Показать, какие файлы будут удалены (безопасный режим)
$ git clean -n
Would remove build/
Would remove temp.log
#### Удалить неотслеживаемые файлы
$ git clean -f
#### Удалить неотслеживаемые файлы и директории
$ git clean -fd
#### Удалить игнорируемые файлы тоже
$ git clean -fx
#### Интерактивный режим
$ git clean -i

Безвозвратно удаляет файлы. ВСЕГДА используй флаг -n для предварительного просмотра
## Сценарии отмены изменений
### Сценарй 1: Исправлений опечатки в неотправленном файле
#### Изменили файл, но ещё не добавили в индекс
$ git status
modified:   src/main.js
#### Отменяем изменения
$ git restore src/main.js

### Сценарий 2: Убрать файл из подготовленного коммита
#### Случайно добавили лишний файл
$ git add .
$ git status
Changes to be committed:
    modified:   src/main.js
    modified:   .env  # Этот файл не должен попасть в коммит

#### Убираем из индекса
$ git restore --staged .env

### Сценарий 3: Полный сброс к чистому состоянию
#### Отменить все изменения в отслеживаемых файлах
$ git restore .

#### Убрать все из индекса
$ git restore --staged .

#### Удалить все неотслеживаемые файлы и папки
$ git clean -fd

### Сценарий 4: Восстановление удаленного файла
#### Файл был удалён в предыдущем коммите
$ git log --oneline -- deleted-file.txt
a5f3b8c Remove obsolete configuration
b7d9e2a Add configuration file

#### Восстанавливаем из коммита, где файл ещё существовал
$ git restore --source=b7d9e2a deleted-file.txt

## Различия между командами
#### git restore
Работает с отслеживаемыми файлами. Восстанавливает как в рабочей директории, так и в индексе
#### git checkout --file
Старый синтаксис для восстановления файлов в рабочей директории
#### git clean
Работает только с неотлеживаемыми файлами, удаляет их безвозвратно

## Меры предосторожности
### Проверка перед отменой
git status git diff для просмтора и изменений перед их отменой
### Создание резервной копии
если изменения могут понадобиться, сохранить их в отдельную ветку перед их отменой
### Осторожность с git clean
ВСЕГДА использовать флаг -n для предварительного просмотра удаляемых файлов

#### Сохранить изменения на всякий случай
$ git stash push -m "Временное сохранение перед отменой"

#### Или создать временную ветку
$ git checkout -b temp-backup
$ git add .
$ git commit -m "Backup before reset"
$ git checkout main
